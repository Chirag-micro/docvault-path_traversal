TASK: shield__docvault-path_traversal_001
CWE-22 Path Traversal via post-validation legacy fallback

ROOT CAUSE
==========

DocVault validates the primary template path before reading:

    templates/<name>/<locale>.html

That path is checked with os.path.commonpath, so direct traversal and
sibling-prefix payloads are rejected. The bug is later in the control
flow. If the primary file is missing, TemplateRenderService.render()
calls _legacy_snapshot_path() for locale-like compatibility tokens such
as:

    legacy.2024.q3.locale_snapshot.preprod

The helper converts that token into:

    templates_archive/retention/2024/q3/locale_snapshot.preprod.html

and the service opens the fallback path without applying the sandbox
predicate to that final path. The code therefore validates one path but
reads another.

EXPLOIT
=======

Request:

    name=invoice
    locale=legacy.2024.q3.locale_snapshot.preprod

Flow:

1. The locale regex accepts the value because it contains only
   alphanumeric characters, dots, underscores, and hyphens.
2. The primary path resolves under templates/invoice and passes
   _validate_in_sandbox().
3. The primary read misses because there is no active template with
   that locale token.
4. The legacy fallback maps the token to templates_archive/retention.
5. FileLoader.read() returns archive bytes from outside the public
   template sandbox.

FIX
===

Validate the final resolved path inside the helper that constructs it,
so _legacy_snapshot_path() never returns a path outside the public
template sandbox:

    resolved = os.path.normpath(os.path.abspath(joined))
    if not self._validate_in_sandbox(resolved):
        return None
    return resolved

This preserves valid template behavior, keeps missing non-legacy
locales as TemplateNotFoundError, and blocks archive reads through the
tenant-facing render API.

WHY THIS IS HARDER
==================

The exploit does not contain "../" and the visible sandbox predicate is
not the vulnerable primitive. Successful agents must trace the late
fallback branch and compare the validated path with the path that is
actually opened.
